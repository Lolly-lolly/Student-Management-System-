# Student-Management-System-
PROJECT NAME: STUDENT MANAGEMENT SYSTEM
The Student Management System is a Python program that:
• Stores student name and matric number
• Displays stored student records
SOFTWARE DEVELOPMENT LIFE CYCLE (SDLC) OF THE PROJECT 
1. Requirement Analysis
The requirement analysis phase defines what the Student Management System should do.
Functional Requirements:
• The system shall accept student name and matric number
• The system shall store student details
• The system shall display student records
Non-Functional Requirements:
• The system should be easy to use
• The system should be reliable

2. System Design
This phase describes how the system will be built.
• Programming Language: Python
• Interface: Command Line Interface
• Data Storage: List
 System Components:
• Student
• Student Management System

3. Implementation
The Student Management System was implemented using Python classes and lists to store student records.

4. Testing
The system was tested by:
• Adding student records
• Displaying student records
• The system worked as expected.

5. Deployment
The project was deployed by:
• Running the program on a local machine
• Uploading the source code to GitHub

6. Maintenance: Maintenance includes fixing bugs and adding new features such as updating or  deleting student records.
